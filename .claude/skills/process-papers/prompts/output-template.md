# Output Template Instructions

Compile all pipeline outputs into a single research briefing document.

## Template

Use this exact structure for the output file:

```
# Research Briefing: {topic}
**Date:** {date}
**Papers reviewed:** {count}
**Depth:** {depth}

---

## Executive Synthesis

{synthesis from Stage 4, or single-paper analysis}

---

## Per-Paper Summaries

{for each paper, include the summary from Stage 2}

### {Paper Title}
{summary content}

**Crosscheck Results:**
{crosscheck results from Stage 3 for this paper}

---

## Cross-Paper Analysis

{if multiple papers}
### Agreements
{agreements from crosscheck Stage 3}

### Contradictions
{contradictions from crosscheck Stage 3}

### Open Questions
{open questions from synthesis}

---

## AI Replication Plans

{for each AI/ML paper, include the replication plan from Stage 5}
{omit this section entirely if no papers were flagged as AI/ML}

### {Paper Title}
{replication plan content}

---

## Study Session Q&A

{for each question asked during the interactive session}

**Q:** {question}
**A:** {answer}

---

## Skipped Papers

{if any papers failed to ingest, list them here}
- {URL or path} — {reason: 404, timeout, corrupted, paywall, etc.}
{omit this section if all papers were successfully ingested}

---

*Generated by /process-papers skill*
```

## File Writing Instructions

1. Determine the output filename: `_briefing-{topic-slug}-{date}.md`
   - Convert topic to lowercase, replace spaces with hyphens
   - Prefix with `_` so briefings sort to the top in Obsidian and are visually distinct from paper files
   - Example: `_briefing-spatial-transcriptomics-2026-03-23.md`

2. Write to the **source folder** (the same folder the papers were read from) using the Write tool.
   - This is the primary output location — the briefing lives alongside the papers it covers.
   - Each run produces a new timestamped file. Previous briefings are NOT overwritten.

3. If `--output` was provided, also write a copy to `{output-path}/{filename}`.

4. Confirm to the user:
   - "Research briefing written to:"
   - "  Source folder: {source-folder-path}/{filename}"
   - "  Local copy: {output-path}/{filename}" (if --output was provided)
